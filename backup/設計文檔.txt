# Eva 設計文檔

## 模塊詳細設計

### interface 模塊
- **功能**：提供統一接口，所有模塊需實現`SelfDescribing` trait。
- **接口**：定義模塊識別和功能聲明。

### manager 模塊
- **功能**：管理插件和流程控制，包括插件加載和命令路由。
- **接口**：提供插件管理和命令分發功能。

### plugin 模塊
- **功能**：提供可擴展功能，如語音合成和互動小說。
- **接口**：定義插件的加載和卸載。

### comms 模塊
- **功能**：負責與外部API的通信，支持本地和遠程模型。
- **接口**：提供通信和數據交換功能。

### state 模塊
- **功能**：管理應用程序狀態和數據持久化。
- **接口**：提供數據存取和狀態管理。

### story_support 模塊
- **功能**：支持故事敘述和沉浸體驗。
- **接口**：提供敘事支持和上下文管理功能。

### ui 模塊
- **功能**：提供用戶界面交互功能。
- **接口**：定義用戶輸入和界面渲染。

## 系統設計更新
- **模塊化設計**：引入模塊化和插件化設計，確保系統靈活性。
- **核心管理模塊**：設計核心AI管理模塊，負責協調和管理各子系統。
- **接口和協議**：使用標準化的通信協議，確保模塊間的有序通信。
- **數據存儲和管理**：集中式數據存儲系統，支持多種數據格式。
- **可擴展性和靈活性**：支持動態添加和移除模塊。
- **監控和日誌**：實現系統監控和日誌功能。

## 2025-05-04 更新

### comms 模塊補充
- 新增 `chat_adapter.rs`：封裝 OpenAI Chat Completions 與本地 LLM 介面。
- 支援模型選擇邏輯：`gpt-4o` / `gpt-4.5-turbo` / 本地模型；讀取環境變數 `EVA_MODEL`。

### LLM 策略
- **邏輯複雜敘事**：預設 `gpt-4o`（若可用）；退而求其次 `gpt-4.5-turbo`。
- **成本/脫線模式**：可指定 `gpt-3.5-turbo` 或本地 Llama。
- 策略中心由 `model_router.rs` 決策，`chat_adapter.rs` 執行實際 API 交換。

### StorySession 設計
- 定義
  ```rust
  struct StorySession {
      id: uuid::Uuid,
      log: Vec<String>,
      created_at: chrono::DateTime<chrono::Utc>,
      model: String,
  }
  ```
- 方法
  - `send_prompt(&mut self, user_msg: &str) -> Result<String>`
  - `save(&self, path: &Path)`
  - `load(path: &Path) -> Result<Self>`
  - `exit(self)`

以上內容納入開發基準文件，後續實裝時請參照。
