# EVA 強制規範（10 條）

1. **嚴格對照藍圖**：任何架構、流程、驗證機制的新增或修改，必須先完整對照現有藍圖（含設計歷史），明確說明依據，嚴禁憑空更動。
2. **禁止未經審查直接修改**：所有核心架構檔案（如本文件、README、設計規範）不得未經討論、審查與記錄就直接更動。
3. **規則條文化**：所有設計原則、驗證流程、例外處理機制，需有明確條文與範例，並收錄於藍圖正式文件。
4. **變更流程必須記錄溯源**：每次架構變更須有明確紀錄、審核流程與溯源依據，並標註於設計歷史檔案。
5. **自動化驗證必須減少人工負擔**：所有自動化、驗證機制須以減少人工為目標，嚴禁為追求形式而增加複雜度。
6. **窗口協作者（含 AI）皆須嚴格遵守**：無論人員或 AI，皆須嚴格依循本規則，不得例外。
7. **違規處理**：若發現有違規或未依規定更動，須立即回溯、修正，並補充記錄原因與處理過程。
8. **[強制] 本地現況同步與窗口資訊一致性**：所有窗口（人員或 AI）在對外溝通、審查、或回應外部建議時，必須先盤點並同步本地最新進度，明確標註哪些已完成、哪些為新需求、哪些屬外部建議，嚴禁僅依藍圖內容而忽略現實狀況，避免資訊落差與誤會。違反時須立即修正並補充記錄。
