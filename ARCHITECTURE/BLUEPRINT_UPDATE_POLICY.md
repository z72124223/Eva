# EVA 藍圖自動分類與同步規範

> 本規範為專案監督與自動化腳本嚴格執行依據，任何窗口、腳本或 AI 助理在本地更新後，**必須自動同步分類更新 ARCHITECTURE 目錄下所有藍圖文件**。

---

## 1. 原則
- 只要任何窗口（人工或自動）對本地檔案做出成功修正，必須立即自動同步至 `ARCHITECTURE/` 目錄下對應藍圖文件。
- 同步時需依據議題自動分類（如 CI、測試、Router、Spec、隱私、Ops…），分別覆蓋/補充對應的 md 文件。
- 任何分類規則、分檔邏輯、段落格式，均需明確記載於本規範，作為自動化與審核依據。

## 2. 自動分類規則
| 議題類型         | 對應藍圖檔案                 | 更新範圍（Section）         |
|------------------|------------------------------|----------------------------|
| CI/CD 覆蓋率     | CI_PR_AUTOMATION.md          | 覆蓋率門檻段落              |
| 測試策略/熔斷    | TESTING.md                   | 測試策略、回滾、熔斷段落    |
| Router 拆分      | ROUTER_DESIGN.md             | 架構拆分、依賴圖            |
| Spec ↔ 程式碼對映| SPEC_MAPPING.md              | 對映腳本、CI 檢查說明        |
| 資料隱私         | PRIVACY_DESIGN.md            | 加解密、權限、隔離設計      |
| Mutation testing | TESTING.md                   | Mutation Testing 段落      |
| Ops/監控         | OPS_DESIGN.md                | SLO、告警、降級策略         |
| 進度/總覽        | CRITICAL_ISSUES_STATUS.md    | 全域總表                    |

- 若議題跨多檔案，需於所有相關檔案同步補充。
- 若檔案尚未存在，腳本必須自動建立並補上標準段落。

## 3. 執行流程
1. 任一窗口（人工/AI/CI）完成本地修正。
2. 腳本自動判斷議題類型，分類補寫/覆蓋對應 md 文件 section。
3. 更新後自動 commit（如於 CI context）或提示人工審核。

## 4. 審核與追蹤
- 每次分類更新皆需於 commit message 註明議題類型與檔案。
- 監督可隨時查閱 `ARCHITECTURE/` 目錄下所有藍圖，確保內容即時、完整、可稽核。

## 5. 違規處理
- 若發現有窗口未依本規範自動分類同步，視為流程違規，需立即補齊並檢討修正。

---

> 本規範為 EVA 藍圖自動化與監督機制核心，所有自動化腳本、AI 助理與人工操作皆必須嚴格遵守。
