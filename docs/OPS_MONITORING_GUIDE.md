# EVA Day-2 Ops 與監控指引

## 1. 關鍵運維指標（建議納入監控/告警）
- **patch 失敗率**（%）：連續失敗/人工介入次數過高自動告警
- **LLM 調用次數**、**錯誤率**：API 5xx/4xx、timeout、quota exceeded
- **平均修補時間**（秒）：異常增高時自動標記
- **自動修補/人工審核比**：判斷自動化成效
- **自動化流程宕機率**：自動重啟/降級模式觸發次數

## 2. 熱修補與降級策略
- LLM API 連錯率 > 5% 時，自動切換為只讀模式（禁自動修補，僅允許查詢/人工審核）
- patch 連續 3 次失敗，自動暫停 30 分鐘並通知維護者
- 監控指標異常時自動產生 issue 或發送告警通知（如 Slack/Email）

## 3. 監控與告警工具建議
- **Prometheus + Grafana**：收集 patch/LLM/CI 指標並視覺化
- **Sentry**：自動追蹤 runtime panic、API error
- **自訂 webhook/alert**：異常時自動通知維護者

## 4. 上線後日常維運流程
- 每日自動產生 patch 成效報告（成功率、人工介入率、平均修補時間）
- 定期審查自動修補/降級觸發紀錄，調整閾值
- 重大異常需人工審核與備份
